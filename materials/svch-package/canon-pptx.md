# Canon · Стилистика боевой презентации СВЧ / ГИС ЦТИ

> **Источник истины:** боевой `ЦТИ_ГИС.pptx` от G (2026-05-08).
> **Назначение этого файла:** чтобы любая новая сессия могла **добавлять слайды** в боевую презу не ломая стилистику, расстояния и ритм.
> **Правило:** прежде чем рисовать слайд — прочитать этот файл. Прежде чем менять канон — обсудить с G.

### Семантика цвета (правило G, не нарушаем)

> **«Позитивные штучки выделяем зелёной плашкой внизу, а проблемы — красной».**

- **КРАСНЫЙ баннер внизу слайда** = диагноз / потеря / боль / методологическая защита (слайды 2, 3)
- **ЗЕЛЁНЫЙ баннер внизу слайда** = новый поток / фискальный эффект / уникальность субъекта / решение (слайды 4, 6, 7)
- **Без баннера** = структурный/обзорный слайд (слайды 5 механика, 8 запрос)
- **Красный акцент в контенте** = проблема, или мобилизующий императив (слайд 8: «САНКЦИОНИРОВАТЬ»)
- **Зелёный акцент в контенте** = новый ответ, новый игрок (слайд 5: «КАСКАДНОЕ ЗАМЕЩЕНИЕ» pill, слайд 6: лид-цифры команды)

**Не путать.** Красный для позитива и зелёный для негатива сломают всю читаемость декка по цвету (а Чемезов листает быстро).

## 2. Шрифты

В боевой `.pptx` использован дисплейный шрифт с характерной геометрией для всех заголовков, больших цифр и caps-надписей. Точное имя шрифта G не зафиксировал явно — шрифт идёт вшитым в `.pptx`.

**Семейство (по визуальной похожести):** что-то из Founders Grotesk Bold / Roobert Bold / похожий гротеск с слегка narrow пропорциями. Body-шрифт похож на тот же гротеск в regular-весе.

**Безопасный fallback при сборке вне Keynote (если по какой-то причине надо собирать новый слайд через python-pptx):**
- **Display (заголовки, цифры):** `Helvetica Neue Bold` → `Arial Black` → системный sans-serif bold
- **Body:** `Helvetica Neue` → `Arial` → системный sans-serif
- **Числовой:** тот же display + `font-feature-settings: "tnum"` (табличные цифры, важно для P&L)

**Запрещено:** Times, любые serif, Courier. Calibri — только как абсолютный fallback.

## 3. Размер слайда и поля

- **Размер:** 16:9 (`13.333" × 7.5"` для PPTX-совместимости; в Keynote — стандарт «Wide»)
- **PAD_X (поля слева/справа):** ~`0.5"`
- **PAD_TOP:** ~`0.42"`
- **PAD_BOT:** ~`0.38"`

## 4. Каркас слайда (chrome)

Каждый контентный слайд (2-8) имеет одинаковую структуру chrome — это держит ритм декка. Новый слайд **обязан** ей следовать.

```
┌──────────────────────────────────────────────────────────────┐
│ ■ КАТЕГОРИЯ                                  ЧАСТЬ N         │  ← top chrome (y≈0.30")
│ ━━━                                                          │  ← красная полоска-акцент (y≈0.55", w≈0.45", h≈0.05")
│                                                              │
│ Заголовок слайда (1-2 строки)                                │  ← H1 (y≈0.92", 32-40pt display bold)
│ Подзаголовок (sub) одной строкой.                            │  ← Sub (y≈1.75", 12-14pt body)
│ ─────────────────────────────────────────────────────────────│  ← чёрная разделительная линия (y≈2.5", 1pt)
│                                                              │
│            [ОСНОВНОЙ КОНТЕНТ]                                │
│                                                              │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━│  ← цветной баннер (опционально, y≈6.55", h≈0.32")
│ ─────────────────────────────────────────────────────────────│  ← bottom rule (y≈6.87", 0.5pt серый)
│ ГИС «ЦТИ» · Стратегическая записка             0N / 09       │  ← footer (y≈6.95")
└──────────────────────────────────────────────────────────────┘
```

### Top chrome

- **Маленький красный квадрат** (~`0.07" × 0.07"`) слева — единственный визуальный анкер сверху
- **Категория слайда** caps-spaced справа от квадрата: «ДИАГНОЗ РЫНКА», «МЕХАНИКА», «ЗАПРОС НА РЕШЕНИЯ» — 10pt bold, серый, letter-spacing 2.4pt
- **Справа: «ЧАСТЬ N»** caps-spaced — 10pt bold, серый dim, letter-spacing 2.4pt. Часть = логический раздел декка (I=диагноз, II=решение, III=механика, IV=инфра+экономика, V=запрос)
- Под top chrome — **тонкая красная полоска** (`0.45"` длиной, `0.05"` толщиной). Единственный декоративный акцент, не убираем

### H1 (заголовок слайда)

- 32-40pt, bold, чёрный (`#111111`)
- Display-шрифт
- Line-spacing 1.05
- 1-2 строки максимум
- **Без знаков препинания на конце**
- **Не использовать caps** — заголовок Capitalized как обычный текст

### Sub (подзаголовок)

- 12-14pt, regular, gray-2 (`#3a362e`)
- Body-шрифт
- **Одна полная фраза, законченная точкой**
- Line-spacing 1.32
- 1-2 строки

### Footer

- Тонкий серый rule (`#c0baae`, 0.5pt) на y≈6.87"
- Слева: `ГИС «ЦТИ» · Стратегическая записка` — 9pt, серый, letter-spacing 1.6
- Справа: `0N` (bold, чёрный) ` / 09` (regular, серый) — 9pt, letter-spacing 1.6
- **Если общее количество слайдов меняется (например, добавляется слайд безопасности) — пересобрать footer везде на новое суммарное количество**

## 5. Контентные паттерны (повторно используемые)

### 5.1. Двухколоночный контраст (слайд 2: диагноз)
- Колонки разделены вертикальной тонкой линией (`#c0baae`, 0.5pt)
- В каждой колонке: eyebrow caps («КОНТУР 1»), затем H2 («Штатный найм + квоты»), затем 2 цифры с подписями, затем eyebrow «ЭФФЕКТ ДЛЯ X» + одна жирная фраза-вывод
- **Цифры 48pt минимум** (если 64pt не помещается — 48, не меньше)

### 5.2. Трёхколоночное расширение (слайд 3: эскалация)
- 3 колонки одинаковые по структуре, разделены вертикальной тонкой линией
- Каждая: eyebrow caps («БАЗА» / «СЕЙЧАС» / «ПРОГНОЗ»), цифра-год (52pt display, цветовой переход жёлтый→оранжевый→красный), карта/визуал, две stats внизу

### 5.3. Трёхколоночное решение с центральной массой (слайд 4: решение)
- Левая и правая колонки **узкие и серые** (мини-описания контуров A и Б)
- Центральная колонка **крупная и яркая** — главный объект (ГИС ЦТИ display 72-84pt)
- Под центральной — тёмная плашка-плейт с двумя метриками
- Снизу — **зелёный баннер** с фискальным эффектом

### 5.4. Карточки-блоки (слайд 5: механика)
- Карточки на белом фоне (`#fff`) с тонкой рамкой `#c0baae`
- В заголовке карточки: огромный номер (01/02 в 64pt красным или зелёным) + label-caps справа от него
- Внутри: H2, pill (плашка с цветом БЫСТРО/МЕДЛЕННО), body
- В боевой добавлен третий блок-карточка под двумя — **РЕПАТРИАЦИЯ КАПИТАЛА**

### 5.5. Таблица P&L (слайд 7: экономика)
- Header row на бежевом-2 (`#e7e1d3`), caps-spaced labels 7.5-8pt
- Data rows на белом, кроме последней (Чистая прибыль) — на лёгком красном (`#fcecea`)
- Цифры 9.5-10pt, тёмные; **последняя строка bold + красный**
- Колонки: первая широкая (название показателя ~2"), остальные узкие (~0.55-0.95")
- **Числа справа выровнены, табличные цифры обязательно**
- Caption под таблицей серым 7.5pt: базовый сценарий + источник

### 5.6. Двухшаговый запрос (слайд 8)
- Огромный номер (01/02 в 80pt красным) слева
- Verb caps-spaced справа от номера («САНКЦИОНИРОВАТЬ» / «ИНИЦИИРОВАТЬ») — 18-20pt display bold
- Под verb: горизонт caps («ГОРИЗОНТ · 1–3 МЕС.») 8.5pt серый
- В правой колонке: тело запроса 14pt, ключевые слова bold + чёрные
- **Без баннера внизу** — слайд сам по себе финальный

### 5.7. Цветной баннер внизу (паттерн для любого слайда)

```
┌──────────────────────────────────────────────────────────────┐
│ Краткий вывод слайда одной фразой, ключевая цифра bold.      │  ← w = SLIDE_W - 2*PAD_X
└──────────────────────────────────────────────────────────────┘  h ≈ 0.32", padding-left 0.2"
```
- **Цвет:** красный `#d72820` (проблема) или зелёный `#7e8f4d` (решение/деньги)
- **Текст:** белый, 10.5-13pt, line-spacing 1.25-1.35, **одна строка** если возможно
- **Y-координата:** ≈ 6.55" (выше — мало воздуха над контентом, ниже — налезает на bottom rule)
- **Если текст не влезает в одну строку** — сократить, не растягивать в две (визуально портит ритм)

## 6. Правила добавления нового слайда

Если в новой сессии Миша/G попросят добавить слайд — следовать этому чек-листу:

1. **Какой слой повествования?** Если новая «Часть» (например, «БЕЗОПАСНОСТЬ» как часть III рядом с «МЕХАНИКА») — оставить ту же ЧАСТЬ N. Если новый слайд внутри существующей категории — увеличить нумерацию слайдов и пересобрать footer везде
2. **Top chrome:** красный квадрат + категория caps-spaced + «ЧАСТЬ N» справа
3. **H1:** одно утверждение, 1-2 строки, без точки, 32-40pt display
4. **Sub:** одна фраза с точкой, 12-14pt серая
5. **Чёрная разделительная линия** на y≈2.5"
6. **Контент:** взять подходящий паттерн из раздела 5 (двух/трёх-колоночный контраст / карточки / таблица / две стопки запросов)
7. **Цвет акцента:** красный если проблема, зелёный если решение/деньги/команда
8. **Баннер:** если есть один сильный вывод — баннер. Цвет по семантике. Текст одной строкой
9. **Footer:** ГИС «ЦТИ» · Стратегическая записка слева, NN / 0[total] справа

## 7. Запреты

- Никаких полупрозрачных слоёв, теней, gradients — кроме тёмной плашки-плейта на слайде 4
- Никаких иконок (кроме маленького красного квадрата в top chrome)
- Никаких округлых рамок (corner-radius = 0 везде, кроме pill-плашек на слайде 5 — там radius ≈ 2pt)
- Никаких изображений-фотографий (только vector-карты/диаграммы, и то — только где это содержит информацию: слайд 3 карты)
- Никаких всплывающих cap fills и больших цветных хедеров наверху слайда — header идёт текстом, без подложки
- **Не путать семантику цвета** (красный для позитива и зелёный для негатива)
- **Не использовать другие цвета акцента** (синий, фиолетовый, жёлтый — кроме оттенка `#e8b61e` для слайда 3 «БАЗА 2019», где это часть triplet жёлтый→оранжевый→красный)

## 8. Шаблон слайда (для нового билда через python-pptx)

Если требуется собрать слайд программно (не через Keynote/PowerPoint руками) — взять как основу мой `build_deck_pptx.py` из предыдущей сессии. Хелперы там готовы: `add_chrome_top`, `add_chrome_bot`, `add_head`, `add_banner`, `add_rect`, `add_line`, `add_text` (с поддержкой letter_spacing через XML и multi-run runs). Цвета и шрифты в начале файла как константы — поменять на актуальные из этого канона.

Но: **первый выбор — собирать слайд руками в PowerPoint/Keynote, копируя существующий слайд боевой и редактируя контент.** Это даёт идеальное соответствие стилю без рисков.
