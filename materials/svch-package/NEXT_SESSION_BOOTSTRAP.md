# BOOTSTRAP для новой сессии Claude (СВЧ/ГИС ЦТИ)

Этот файл — что Claude должен сделать **первым делом** в новой сессии.

---

## Шаг 1. Прочитать контекст в этом порядке

1. `materials/svch-package/HANDOFF.md` — где остановились, критерии оценки вариантов ответов, карантин имён.
2. `materials/svch-package/REQUESTS.md` — 6 запросов целиком (что G отправлял в Perplexity/Gemini).
3. `materials/svch-package/canon-pptx.md` — стиль и структура боевой презы.
4. Репо `grantik/odin-vault`: `canon/labor-market.md`, `canon/legal-references.md` — текущий канон фактов.

## Шаг 2. Прочитать boevoy xlsx

G приложит `ГИС_ЦТИ__модель_оператора__налоги__дэшборд_.xlsx` к новой сессии. Файл будет в `/mnt/user-data/uploads/`. Использовать `view` или Python (через bash_tool: `pip install openpyxl --break-system-packages`).

3 листа: Вводные / По месяцам / Ставки по профессиям. Понять структуру и формулы — это база, к которой потом делается аддендум (CAPEX 90→120, новый лист `Y1_Monthly_Plan`).

## Шаг 3. Получить от G варианты ответов на запросы

G присылает по запросу 1 (потом 2, потом возможно остальные) — **3 варианта ответа** от разных моделей/конфигураций. Задача:

1. Сравнить 3 варианта по критериям из HANDOFF.md (главный критерий — **полнота ссылок на первоисточники**).
2. Дать вердикт: какой выбираем, почему, или "все три недостаточны — переделать".
3. После запроса 1 (или 2) рекомендовать G: продолжать ли в той же связке моделей для запросов 3–6, или менять.

## Шаг 4. Извлечение данных из выбранного варианта

Из победившего варианта:
- Цифры с датами и URL → пополнить `odin-vault/canon/labor-market.md` или `legal-references.md`.
- Цитаты с датами и URL → отдельный файл `materials/svch-package/quotes-bank.md`.
- URL первоисточников → таблица для футера/приложения презы (`materials/svch-package/sources-bank.md`).

## Шаг 5. Только когда данные собраны — обновлять презу

Правки слайдов и аддендум к xlsx делаются **после** того как закрыта дыра по источникам. Не раньше.

---

## Что НЕ делать

- Не пытаться загружать xlsx в репо через `GTHB:create_or_update_file` или `push_files` — MCP-обёртка ломает бинарь. xlsx живёт только в uploads текущей сессии.
- Не нарушать карантин имён (см. HANDOFF.md).
- Не давать цифры без источников — это сейчас главная дыра в подаче, её закрываем именно сейчас.
- Не менять роль: Claude = оркестратор, не исполнитель CC. Готовые промпты для CC/Perplexity G копирует целиком.
