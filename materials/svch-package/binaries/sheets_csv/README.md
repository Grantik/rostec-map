# CSV-дампы листов боевого xlsx

Источник: `ГИС_ЦТИ__модель_оператора__налоги__дэшборд_.xlsx` (boevoy, со стороны G).

Дампы сделаны для того, чтобы любая будущая сессия Claude/CC могла прочитать **структуру и значения** боевой модели через GTHB API без необходимости иметь сам xlsx-бинарь в контексте.

## Листы (9)

| Лист | Что в нём |
|---|---|
| `О_модели.csv` | Архитектура, легенда цветов, методология фиксала, источники |
| `DASHBOARD.csv` | Единственный лист с входными параметрами (4 рычага: H, R, регион, stress-test ОПС) |
| `PARAMS.csv` | Фиксированные косты, налоги, региональные косты, ставки. **CAPEX_per_new = 90 000 ₽** (TODO: поднять до 120 000) |
| `Unit_Economics.csv` | P&L на 1 уч./мес в 4 сценариях + базовый из DASHBOARD |
| `Escrow_Balance.csv` | Эскроу как балансовое обязательство, float-доход в РНКО |
| `TCO_Comparison.csv` | Дельта ГИС vs штатник в %, со sticky-эффектом |
| `Fiscal_7Y.csv` | Налоги 3-уровневого каскада на 7 лет, номинал + реал |
| `Operator_PnL_7Y.csv` | P&L и Cash Flow Оператора, 7-летняя траектория |
| `Sensitivities.csv` | Сетка маржи EBITDA по H × R |

## Важное

Дампы — это **values only** (значения формул на момент дампа), не сами формулы. При желании выполнить пересчёт модели (например, поднять CAPEX 90→120) — это надо делать в исходном xlsx через openpyxl c `data_only=False` или вручную в Sheets. Дампы — для grep, чтения, ссылок в memo.

Исходный xlsx-бинарь — в Drive (см. parent README).
