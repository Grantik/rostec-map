## Что в презу НЕ ИДЁТ без верификации

- **Литературные категории «гражданская смерть», «управляемая прозрачность», «цифровое принуждение к законности»** (V2) — собственные обобщения модели, не нормы из актов. Не использовать.
- **URL `kremlin.ru/acts/bank/52331` для ФЗ-289 (V2)** — формат правдоподобный, но конкретный URL должен быть проверен через web_fetch. Если не подтвердится — использовать `pravo.gov.ru/Document/View/0001202507310020` или `kremlin.ru/acts/news/77694`.
- **«688 тыс. проверок в реестре за 2 дня» (V2)** — количество проверок ≠ численность реестра, цифра без первоисточника, не использовать.
- **Региональные карточки ОКВЭД-запретов в V2** (Калужская как «модель», сводная таблица Тюменская/Тверская/Воронежская/Калужская) — частично пересекаются с Q2 verdict (МО ПП №390-ПГ, СПб №89-пг, НСО ПП №184, Свердловская Указ №93-УГ, Ростовская Указ февраль 2026); использовать данные из Q2, не V2.
- **«2024 ~156 000» как общая квота ИРС (V2, Карточка 7)** — без прямого URL ПП за 2024; в стартере V1 точная цифра не приведена; нужна сверка через government.ru/docs/ за 2024 (если такая публикация была — это ПП от ноября/декабря 2023).
- **«Указ №42 от 30.01.2024 (поправки в концепцию миграционной политики до 2025)»** — не отражён ни в одном из 3 вариантов; пометить как «нужна верификация».
